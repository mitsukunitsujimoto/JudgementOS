# JudgmentOS — ブランドガイド

## ブランド体験（最優先）

JudgmentOS を開いた瞬間、**「これはChatGPTではない」** と感じること。

OS は入力から始まらない。**世界観から始まる。**

### 最初の画面（Version 1.2）

```
JudgmentOS
判断の質を支える思考OS

判断の質は、
判断文脈の質で決まります。
置くのは、本人だけです。

JudgmentOSは、
判断文脈を言語化し、その質を高め続ける
ための思考OSです。

【判断文脈を置く】

答えを出すツールではありません。
```

### 核（Version 1.2）

**JudgmentOSは、判断文脈を言語化し、その質を高め続ける思考OSである。**

順序は常に：本人 → 判断文脈 →（必要なら）問い → 文脈の更新 → 判断。

成功とは便利さではない。「何を渡せていたか」に気付き、「判断文脈に一文を足す」と決めること。

### 体験の流れ · Version 1.2

```
①〜⑤ 意思の言語化（気になる → テーマ → 実現 → 守る → 映し返し → 制約）
  ↓
判断文脈を見る（渡す前に、置いたものを確認）
  ↓
まだ言葉になっていない層への問い（固定一覧ではない · 任意スキップ可）
  ↓
今回、何を渡せていなかったか → 判断文脈に一文を足す（または今回はこのまま渡す）
  ↓
部下への問い返しの型
  ↓
（任意）私はこう判断する → 余韻（判断文脈は育つ）
```

トレードオフは独立の問いではない。③と④を並べるだけ。命名しない。

### 画面を閉じる瞬間

持ち帰るものは **AIの答えではない。あなた自身の判断** である。

## 共通思想（講演 · LinkedIn · 書籍 · プログラム共通）

> AIは最適な答えを示すことはできる。  
> しかしどの未来を目指すかは決められない。

この一文は JudgmentOS の思想そのもの。画面の最初か最後に必ず表示する。

## 使命

AI を賢くすることではない。

**経営者が、昨日より深く考えられるようになること。**

## JudgmentOS とは何か

- 答えを出すツールではない
- 問いを作るツールでもない
- AI へ相談するツールでもない

**経営者が、自分自身の意思決定を映し、未来を決めるための OS。**

## 三層構造（開発者向け · 体験後に理解される）

```
JudgmentOS                    ← 判断プロセス全体を支える OS
│  Decision Context Mirror     ← 中核機能
│    └ 問いの鏡                ← 入口ワークスペース
```

## 判断の一直線

```
Premise → Decision Context → Question → AI → Judgment
```

主役は **Judgment（意思決定）**。Decision Context は Judgment を支える構造。

## 言葉の使い方

| 場面 | 表記 |
|------|------|
| ランディング CTA | **映してみる** |
| 入口 | **気になっていること**（判断から始めない） |
| AIへ渡すもの | テーマ · 実現 · 守る · 制約 · 本人の言葉の映し返し の5点のみ |
| 禁止 | 「トレードオフ」「診断」「分析」「命名」 |
| 最初から | 「文脈を映す」は使わない。意思は体験の中で言語化する |

## UI 表記ルール

| 位置 | 表記 |
|------|------|
| 1画面目 | 世界観のみ。入力欄は出さない |
| 2画面目以降 | 判断入力 → 判断を映す |
| ワークスペースヘッダー | `JudgmentOS` のみ（技術用語を出さない） |
| 締め | 思想の余韻。DB設計は折りたたみ |
