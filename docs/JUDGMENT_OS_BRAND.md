# JudgmentOS — ブランドガイド

## ブランド体験（最優先）

JudgmentOS を開いた瞬間、**「これはChatGPTではない」** と感じること。

OS は入力から始まらない。**世界観から始まる。**

### 最初の画面（Version 1.3 · モニター）

トップは思想語彙より **使ってみたい** を優先する。「判断文脈」は体験の途中で紹介する。

```
JudgmentOS
AI時代の判断を支える思考OS

良い判断は、
AIに答えを聞く前に始まります。
自分が何を大切にしているかを、
まず言葉にしてみませんか。

答えを出すツールではありません。
判断の前に、考えを整えるための場所です。

【判断の準備を始める】

3〜5分で、いま大切なことを言葉にできます。
```

体験中（置いた考えを見た瞬間）で自然に紹介する：

> JudgmentOSでは、これを判断文脈と呼びます。

### 核（Version 1.3）

**JudgmentOSは、AIとの対話を通じて、自分自身の判断基準を育てる思考OSである。**

価値はAIの回答ではない。判断文脈が育ち、自分の判断基準が更新されること。

順序は常に：本人 → 判断文脈 →（必要なら）問い → 文脈の更新 → 判断基準の言語化。

成功とは便利さではない。「AIの答えが変わった」ではなく、「自分の判断基準が変わった」と実感すること。

循環：考えている → AIへ渡す → AIから戻る → 育てる（文脈を残す → 判断基準を未来の自分へ残す）

### 体験の流れ · Version 1.3

```
考えている（①〜⑤・映し返し・問い・気づき・次はこう渡す）
  ↓
AIへ渡す（文面コピー）
  ↓
AIから戻る（任意貼付＋自分に戻る3問）
  ↓
育てる（渡す前／後を見比べて判断文脈を残す）
  ↓
JudgmentOSに向き合う前と後で、判断はどう変わったか（振り返り）
  ① 最初の私は（文脈は編集不可）＋あの時どう考えていたか
  ② 今の私は、どう考えるようになったか
  ③ 今回、一番変化した判断基準は何か
  ↓
この判断基準を未来の自分へ残す
  ↓
一段深くなった（任意：私はこう判断する／これまで育てた判断文脈と基準）
```

問い画面は読むだけで終わらせない。気づきを一文残し、それを見ながら判断文脈を更新する。

V1.3 のクライマックスは、**入力画面ではなく変化を振り返る画面**である。「自分は何が変わったのか」を言葉にし、未来の自分へ残す。「問い返す型」は標準フローには含めない。

トレードオフは独立の問いではない。③と④を並べるだけ。命名しない。

### 画面を閉じる瞬間

持ち帰るものは **AIの答えではない。あなた自身の判断基準** である。

## 共通思想（講演 · LinkedIn · 書籍 · プログラム共通）

> AIは最適な答えを示すことはできる。  
> しかしどの未来を目指すかは決められない。

この一文は JudgmentOS の思想そのもの。画面の最初か最後に必ず表示する。

## 使命

AI を賢くすることではない。

**経営者が、昨日より深く考えられるようになること。**

## JudgmentOS とは何か

- 答えを出すツールではない
- 問いを作るツールでもない
- AI へ相談するツールでもない

**経営者が、自分自身の意思決定を映し、未来を決めるための OS。**

## 三層構造（開発者向け · 体験後に理解される）

```
JudgmentOS                    ← 判断プロセス全体を支える OS
│  Decision Context Mirror     ← 中核機能
│    └ 問いの鏡                ← 入口ワークスペース
```

## 判断の一直線

```
Premise → Decision Context → Question → AI → Judgment Criteria
```

主役は **本人の判断基準の更新**。Decision Context は Judgment を支える構造。

## 言葉の使い方

| 場面 | 表記 |
|------|------|
| ランディング CTA | **映してみる** / **判断の準備を始める** |
| 入口 | **気になっていること**（判断から始めない） |
| AIへ渡すもの | テーマ · 実現 · 守る · 制約 · 本人の言葉の映し返し の5点のみ |
| 締め | **判断基準**（AI回答の保存ではない） |
| 禁止 | 「トレードオフ」「診断」「分析」「命名」 |
| 最初から | 「文脈を映す」は使わない。意思は体験の中で言語化する |

## UI 表記ルール

| 位置 | 表記 |
|------|------|
