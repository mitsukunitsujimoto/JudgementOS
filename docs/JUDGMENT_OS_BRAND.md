# JudgmentOS — ブランドガイド

## ブランド体験（最優先）

JudgmentOS を開いた瞬間、**「これはChatGPTではない」** と感じること。

OS は入力から始まらない。**世界観から始まる。**

### 最初の画面

```
JudgmentOS
経営者の意思決定を支えるOS

AIは最適な答えを示すことはできます。
しかしどの未来を目指すかは決められません。
未来を決めるのは経営者です。

JudgmentOSは、未来を決める前の
あなたの意思決定文脈を映します。

【映してみる】

これはChatGPTではありません。
```

### 体験の流れ（最終成果物 = 新しい判断）

```
判断を書く
  ↓ 前提が映る
  ↓ 見えていなかったものに気づく
  ↓ 意思決定文脈を書き換える
  ↓ 新しい問いが生まれる
  ↓ 必要ならAIへ相談する
  ↓ 私はこう判断する
  ↓ あなたの新しい判断  ← 最終成果物
```

JudgmentOS は Decision Context を作るツールではない。Decision Context を通して、経営者が**新しい判断**を行うための OS。

### 画面を閉じる瞬間

持ち帰るものは **AIの答えではない。あなた自身の判断** である。

## 共通思想（講演 · LinkedIn · 書籍 · プログラム共通）

> AIは最適な答えを示すことはできる。  
> しかしどの未来を目指すかは決められない。

この一文は JudgmentOS の思想そのもの。画面の最初か最後に必ず表示する。

## 使命

AI を賢くすることではない。

**経営者が、昨日より深く考えられるようになること。**

## JudgmentOS とは何か

- 答えを出すツールではない
- 問いを作るツールでもない
- AI へ相談するツールでもない

**経営者が、自分自身の意思決定を映し、未来を決めるための OS。**

## 三層構造（開発者向け · 体験後に理解される）

```
JudgmentOS                    ← 判断プロセス全体を支える OS
│  Decision Context Mirror     ← 中核機能
│    └ 問いの鏡                ← 入口ワークスペース
```

## 判断の一直線

```
Premise → Decision Context → Question → AI → Judgment
```

主役は **Judgment（意思決定）**。Decision Context は Judgment を支える構造。

## 言葉の使い方

| 場面 | 表記 |
|------|------|
| ランディング CTA | **映してみる** |
| 判断入力後の実行 | **判断を映す** |
| ローディング | 判断を映しています… |
| ⑥のブロック（体験後） | あなたの判断の構造 → ここに意思決定文脈が映る |
| 最初から | 「文脈を映す」は使わない。文脈は体験して初めて理解される |

## UI 表記ルール

| 位置 | 表記 |
|------|------|
| 1画面目 | 世界観のみ。入力欄は出さない |
| 2画面目以降 | 判断入力 → 判断を映す |
| ワークスペースヘッダー | `JudgmentOS` のみ（技術用語を出さない） |
| 締め | 思想の余韻。DB設計は折りたたみ |
