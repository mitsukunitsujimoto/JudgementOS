JudgmentOSとは

JudgmentOSは、AIとの対話を通じて、自分自身の判断基準を育てる思考OSです。

順序は常に：本人 → 判断文脈 →（必要なら）問い → 文脈の更新 → 判断基準の言語化。

価値はAIの回答ではありません。判断文脈が育ち、自分の判断基準が更新されることです。

判断文脈も判断基準も、一度作れば終わりではありません。判断するたびに育っていきます。

体験の循環（Version 1.3）

考えている → AIへ渡す → AIから戻る → 育てる

目的は「AIの答えが変わった」ではなく、「自分の判断基準が変わった」と実感することです。

  1. 自分と向き合い、判断文脈を言語化する
  2. AIへ渡す文面をコピーし、任意のAIと対話する
  3. AIの答えを受けて、自分に戻る3問で確かめる
  4. 渡す前と後を見比べ、育てた判断文脈を残す
  5. 判断の変化を振り返り（あの時／今／一番変わったこと）、判断基準を未来の自分へ残す
  6. 後から、育てた判断文脈と今回育った判断基準を開き直せる

良い判断は、AIに答えを聞く前に始まります。

答えを出すツールではありません。判断の前に、考えを整えるための場所です。

ボタン：判断の準備を始める

3〜5分で、いま大切なことを言葉にできます。

画面① · 気になっていること

問い：今、仕事や経営で気になっていることは何ですか。

まだ判断にしなくて大丈夫です。今、気になっていることを書いてください。

操作：入力 → 次へ（一つのテーマに集中）

画面② · 今日のテーマ

問い：その中で、今日いちばん深く考えたいテーマはどれですか。

一つ選びます。（気になっていることが一つのときは自動で進みます）

画面③ · 実現したいこと

問い：そのテーマについて、あなたは何を実現したいですか。

解決策ではなく、向かいたい方向を、あなたの言葉で書きます。

画面④ · 守りたいもの

問い：それを進める中で、何を守りたいですか。

人・関係・時間・信頼・文化など。崩したくないものを書きます。

画面 · 映し返し

あなたは「（実現したいこと）」と考えています。
一方で、「（守りたいもの）」と考えています。
この二つを両立させながら考えることが、今回のテーマになりそうです。

トレードオフなどの命名はしません。本人の言葉を並べるだけです。

画面⑤ · 条件・制約

問い：このテーマで無視できない条件や制約は何ですか。

予算・人数・期限・規程など。

ボタン：いま言葉にしたものを見る

画面 · いま言葉にしたものを見る（判断文脈）

JudgmentOSでは、いま言葉にしたことを判断文脈と呼びます。

  • 今日のテーマ
  • 実現したいこと
  • 守りたいもの
  • 無視できない条件

判断の質は、この判断文脈の質で決まります。
この文脈が、AIや他者へ渡す前提になります。

操作：まだ言葉になっていない層へ進む / 判断文脈を更新する / 判断文脈をコピー（任意）

画面 · まだ言葉になっていない判断文脈へ

正解はありません。全部に答える必要もありません。
気になった問いだけ立ち止まり、一文でも言葉が増えれば十分です。

各問いの下に入力欄「この問いで気づいたこと」（一文で）があります。

問いの例（毎回同じ一覧にはなりません。置いた言葉から動的に選ばれます）：

  • この実現が果たされたとき、自分以外の誰の何が変わっている想定ですか。それは言葉になっていますか。
  • 守ろうとしているのは、制度・数字・それとも特定の人との関係ですか。
  • この制約は、変えられない事実ですか。自分が課した上限ですか。それとも、まだ確かめていない仮定ですか。
  • 一年後に同じ言葉で振り返ったとき、今の実現と守るは、同じ重みで残っていますか。
  • この判断の成功を、最終的に誰が認める想定ですか。それは渡す文脈に入っていますか。

ボタン：判断文脈を育てる

画面 · 次は、こう渡す。

問いから残した「この問いで気づいたこと」を見ながら、次に渡す判断文脈へ足す一文を整えます。

見出し：次は、こう渡す。

操作：この一文で判断文脈を育てる / 今回はこのまま渡す

必要なら、気づきをAIで一文に統合するための依頼文をコピーできます。採用するのは本人です。

画面 · 判断の変化を振り返る（Version 1.3 · クライマックス）

タイトル：JudgmentOSに向き合う前と後で、あなたの判断はどう変わりましたか。

AIの回答を書く場所ではありません。
あなた自身の考え方、判断基準、立ち位置がどのように変わったかを振り返り、言葉にします。

  1. 最初の私は — 最初のテーマ・判断文脈をそのまま表示（編集不可）。その下で「あの時の私は、どう考えていたか。」
  2. 今の私は — 「今の私は、どう考えるようになったか。」
  3. 一番大きく変わったこと — 「今回、一番変化した判断基準は何ですか。」

ボタン：この判断基準を未来の自分へ残す

画面 · 私はこう判断する（任意）

持ち帰るものは、答えではなく、あなた自身の判断です。

操作：判断を残す / 判断は後で · 余韻へ

画面 · 余韻

判断基準が、一段深くなりました。

AIの答えが変わったのではありません。
あなた自身の判断基準が更新されました。

核となる思想

  • 主役はAIではなく、本人の判断です。
  • JudgmentOSは答えを出しません。判断文脈と、自分の判断基準を育てます。
  • 「判断文脈」という語は、体験の途中で自然に紹介します。
  • 固定の観点カタログを製品の顔にしません。
